# Source Audit

Use this skill before giving a final answer based on web research, especially when facts are recent, contested, expensive to act on, or easy to misread.

## Workflow

1. Record the source type, publisher, canonical URL, date, and version or release identifier when present.
2. Prefer official sources first and cross-check unstable claims with an independent second source.
3. Separate direct evidence from your own inference.
4. Deduplicate mirrors, press rewrites, SEO copies, and repeated forum summaries.
5. Flag missing dates, stale pages, ambiguous ownership, or unresolved contradictions.

## Rules

- Use absolute dates for time-sensitive facts.
- Quote minimally and summarize in your own words when possible.
- If a recommendation depends on popularity or community uptake, say what evidence was used and what it does not prove.
- If primary sources are incomplete, state the gap instead of filling it from memory.

## Output

- A compact evidence table or bullet list with quality notes.
- A clear split between confirmed facts, supported inference, and unresolved uncertainty.
