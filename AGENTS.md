# Project Air Defense: AI Agent Guide

Keep this file small. Route outward instead of duplicating detail.

## Mandatory load order
1. `AGENTS.md`
2. `docs/reference/ai-agent-context-routing.md`
3. `docs/reference/ai-agent-skills.md`
4. `docs/index.md` when the routing doc alone is not enough
5. one nearest domain doc, one nearest code file, and one nearest test

Do not broad-read the repo on first pass.

## Mandatory efficiency rules
- Be concise by default in live AI communication only. Never use `caveman`-style writing in repo docs, README prose, or function descriptions.
- Search first, patch second, benchmark third.
- Open at most 4 task-specific files on the first pass.
- Run at most 2 refine loops before naming the edit target and verification lane.
- Add at most 3 files per escalation round.
- Hard stop at 8 task-specific files; summarize before opening more.
- Keep changes deterministic, explicit, and reproducible.
- Do not use background agents or subagents unless the user explicitly asks for them.
- Do not invent licenses, benchmarks, taps, screenshots, or device behavior, and do not paste PATs, passwords, or internal-only URLs into external tooling.
- Use repo-local skills in `.agents/skills/` before falling back to generic habits.
- Update docs and `android/assets/ATTRIBUTION.md` whenever workflow assumptions or external assets change.
- Prefer focused tests and narrow verification lanes before broad runtime checks.
- Pin versions intentionally. Do not move dependencies or workflows to floating `latest`.
- The README badge row is generated by `tools/update_readme_badges.py`.
- Repo-local caveman behavior is pinned to upstream `v1.5.0`; upstream adds configurable default mode, `off`, and `/caveman-help`, while this repo keeps only the mandatory instruction-only overlay: no hooks, no flag files, no session-persistence writes.

## Repository map
- `core/src/main/kotlin/com/airdefense/game/StartScreen.kt`: menu flow and touch targets
- `core/src/main/kotlin/com/airdefense/game/BattleScreen.kt`: 3D scene, HUD, VFX, imported assets, battle loop
- `core/src/main/kotlin/com/airdefense/game/BattleSimulation.kt`: shared simulation core
- `core/src/main/kotlin/com/airdefense/game/BattleLogic.kt`: pure gameplay math helpers
- `android/src/main/java/com/airdefense/game/AndroidLauncher.kt`: Android launch, compatibility, quality mode
- `scripts/run-battle-monte-carlo.*`: headless balance sweeps using the same simulation as the GUI
- `scripts/run-benchmark-suite.*`: report-oriented benchmark runner
- `docs/`: indexed architecture, benchmark, release, AI workflow, and asset-source docs
- `.agents/skills/`: repo-local agent overlays

## External-asset rules
- Prefer explicit CC0 or MIT assets for shipped runtime content.
- Treat open city datasets as reference or layout inputs until their shipping license is verified.
- Record every imported model, texture, or reference image in `android/assets/ATTRIBUTION.md`.

## Verification minimum
- Formatting/lint: `./gradlew ktlintCheck`
- Logic changes: `./gradlew :core:test`
- Balance changes: `./gradlew :core:runBattleMonteCarlo -Pruns=300 -Pwaves=1 -Pseconds=48 -Pstep=0.05`
- Android behavior changes: `./gradlew :android:installDebug`
- Repo badge/doc workflow changes: `python3 tools/update_readme_badges.py --check`

## Done means
- exact files changed are known
- exact verification level is stated
- docs match behavior
- repo stays clean after generated artifacts are removed or ignored
